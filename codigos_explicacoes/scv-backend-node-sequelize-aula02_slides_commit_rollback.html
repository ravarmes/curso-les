Quanto maior o número de tabelas e relacionamentos no mesmo caso de uso, maior a necessidade de commit/rollback.

Código 1 de 3

UfService.create (objeto simples)

UfService.create 
static async create(req) {
  const { sigla, nome } = req.body;
  const obj = await Uf.create({ sigla, nome });
  return await Uf.findByPk(obj.id, { include: { all: true, nested: true } });
}

Visualmente simples

É um registro principal sem lista de dependências para gravar junto.

Sem cadeia de inserts

Não há hasMany ou belongsToMany dentro do mesmo create.

Mensagem didática

Objeto simples ajuda a contrastar com os compostos.

Código 2 de 3

ClienteService.create (hasMany Telefone)

ClienteService.create 
static async create(req) {
  const { nome, cpf, rua, numero, debito, nascimento, bairro, telefones } = req.body;
  if (bairro == null) throw 'O Bairro do Cliente deve ser preenchido!';
  const t = await sequelize.transaction();
  const obj = await Cliente.create(
    { nome, cpf, rua, numero, debito, nascimento, bairroId: bairro.id },
    { transaction: t }
  );
  try {
    await Promise.all(telefones.map(telefone => obj.createTelefone(telefone, { transaction: t })));
    await t.commit();
    return await Cliente.findByPk(obj.id, { include: { all: true, nested: true } });
  } catch (error) {
    await t.rollback();
    throw 'Problema na inserção de um dos telefones do cliente!';
  }
}

Relação 1:N

Cliente precisa nascer com todos os telefones.

Commit no sucesso total

Confirma apenas depois de inserir todos os itens.

Rollback em qualquer falha

Evita cliente salvo sem a coleção completa.

Código 3 de 3

FilmeService.create (hasMany + belongsToMany)

FilmeService.create 
static async create(req) {
  const { titulo, genero, duracao, imagem, tipoDeFilme, diretores, participacoes } = req.body;
  if (tipoDeFilme == null) throw 'O Tipo de Filme do Filme deve ser preenchido!';
  const t = await sequelize.transaction();
  const obj = await Filme.create(
    { titulo, genero, duracao, imagem, tipoDeFilmeId: tipoDeFilme.id },
    { transaction: t }
  );
  try {
    await Promise.all(participacoes.map(participacao =>
      obj.createParticipacao({ personagem: participacao.personagem, artistaId: participacao.artista.id, filmeId: obj.id }, { transaction: t })
    ));
    await Promise.all(diretores.map(diretor => obj.addDiretores(Diretor.build(diretor), { transaction: t })));
    await t.commit();
    return await Filme.findByPk(obj.id, { include: { all: true, nested: true } });
  } catch (error) {
    await t.rollback();
    throw 'Um dos artistas ou diretores informados não foi encontrado!';
  }
}

Duas coleções relacionadas

Participações (hasMany) + Diretores (belongsToMany).

Maior complexidade

Sem transação, o filme pode ficar parcialmente montado.

Mesma estratégia

Uma transação única para todas as gravações do caso de uso.

Fluxo visual

Quando executar commit e rollback

Passo 1

Abre transação

Inicia o contexto comum.

Passo 2

Grava principal

Cliente/Filme nasce na transação.

Passo 3

Grava dependências

Telefones, participações, diretores.

Passo 4

Decisão

commit ou rollback.

Sucesso completo

  • Commit confirma o pacote inteiro.
  • Estado final consistente.

Falha em qualquer etapa

  • Rollback desfaz tudo.
  • Estado anterior é preservado.
