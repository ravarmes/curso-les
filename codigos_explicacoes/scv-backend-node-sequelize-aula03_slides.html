Atividade 09: Implementar - Back-End (Casos de Uso)

Tema: Videolocadora

O sistema contempla os seguintes processos principais:

  • Empréstimo: registro do empréstimo de fitas, vinculado a um pagamento obrigatório (sem uso de créditos).
  • Reserva: permite ao cliente reservar fitas, podendo haver restrições caso não efetive reservas com frequência.
  • Devolução: registro da devolução das fitas; atrasos geram multa e suspensão do cliente até a regularização.
  • Pagamento de multa: cálculo automático de valores (incluindo multas e descontos) e registro manual do pagamento, necessário para liberar novos empréstimos ou remover pendências.

Foco da Apresentação: Processo de Empréstimo de Fitas.

🔄 Fluxo de Transação

💻 EmprestimoService.js (Create)

class EmprestimoService {
  static async create(req) {
    const { data, valor, cliente, itens } = req.body;
    
    // Validação
    if (await this.verificarRegrasDeNegocio(req)) {
      
      // Início da Transação
      const t = await sequelize.transaction();
      
      const obj = await Emprestimo.create(
        { data, valor, clienteId: cliente.id }, 
        { transaction: t }
      );
      
      try {
        await Promise.all(itens.map(item => 
          obj.createItem({ ..., emprestimoId: obj.id, fitaId: item.fita.id }, { transaction: t })
        ));
        
        await Promise.all(itens.map(async item => 
          (await Fita.findByPk(item.fita.id)).update({ disponivel: 0 }, { transaction: t })
        ));
        
        // Finaliza Transação
        await t.commit();
        return await Emprestimo.findByPk(...);
        
      } catch (error) {
        // Desfaz Transação
        await t.rollback();
        throw "Pelo menos uma das fitas não foi encontrada!";
      }
    }
  }
}

static async verificarRegrasDeNegocio(req) {
  
  // ==========================================
  // Regra de Negócio 1
  // ==========================================
  const devedores = await ClienteService.findDevedores();
  // ... lógica para checar se cliente está nos devedores
  if (clienteDevedor) throw "Cliente deve multas!";
  
  // ==========================================
  // Regra de Negócio 2
  // ==========================================
  for (let item of itens) {
    const reserva = await ReservaService.findByFitaStatusAndOutroClienteRN(item.fita.id, '0', cliente.id);
    if (reserva.length != 0) fitasReservadas = true;
  }
  if (fitasReservadas) throw "Fitas com reservas para outros clientes!";
  
  // ==========================================
  // Regra de Negócio 3
  // ==========================================
  for (let item of itens) {
    const fita = await FitaService.findByIdAndDanificada(item.fita.id, true);
    if (fita.length != 0) fitasDanificadas = true;
  }
  if (fitasDanificadas) throw "Fitas danificadas selecionadas!";
  
  return (!clienteDevedor && !fitasReservadas && !fitasDanificadas);
}

Regra de Negócio 1: Cliente sem multas não pagas. O método abaixo usa o Sequelize para executar uma consulta crua (Raw Query).

💻 ClienteService.js

static async findDevedores() {
    const objs = await sequelize.query(
        "SELECT clientes.* FROM emprestimos INNER JOIN clientes ON emprestimos.cliente_id = clientes.id INNER JOIN multas ON emprestimos.id = multas.emprestimo_id WHERE multas.pago = false", 
        { type: QueryTypes.SELECT }
    );  
    return objs;
}

⚙️ SQL Resultante (SQL tradicional)

O ORM processa o comando e executa nativamente o seguinte SQL no banco de dados:

SELECT clientes.* 
FROM emprestimos 
INNER JOIN clientes ON emprestimos.cliente_id = clientes.id 
INNER JOIN multas ON emprestimos.id = multas.emprestimo_id 
WHERE multas.pago = false;

Implementação da Regra 2 📼 Processo: Empréstimo

05

Regra de Negócio 2: Fitas reservadas para outros clientes não podem ser emprestadas. A query injeta os parâmetros de forma segura no Sequelize.

📊 Visualizar Tabelas do Banco de Dados

💻 ReservaService.js

static async findByFitaStatusAndOutroClienteRN(fitaId, status, clienteId) {
    const objs = await sequelize.query(
        "SELECT * FROM reservas WHERE reservas.fita_id = :fitaId AND reservas.status = :status AND reservas.cliente_id != :clienteId", 
        { replacements: { fitaId: fitaId, status: status, clienteId: clienteId }, type: QueryTypes.SELECT }
    );
    return objs;
}

⚙️ SQL Resultante (SQL tradicional)

Após a injeção dos parâmetros (:fitaId, :status e :clienteId), o banco recebe o comando similar a: 

SELECT * FROM reservas 
WHERE reservas.fita_id = 1 AND reservas.status = '0' AND reservas.cliente_id != 1;

Implementação da Regra 3 📼 Processo: Empréstimo

06

Regra de Negócio 3: Fitas danificadas não podem ser emprestadas. O framework ORM gera a consulta SQL pelo método findAll.

📊 Visualizar Tabelas do Banco de Dados

💻 FitaService.js

static async findByIdAndDanificada(id, danificada) {
    const objs = await Fita.findAll({ 
        where: { id: id, danificada: danificada } 
    });
    return objs;
}

⚙️ SQL Resultante Gerado (SQL tradicional)

A partir do objeto JavaScript o Sequelize escreve e despacha para o banco:

SELECT id, danificada, disponivel, created_at, updated_at, filme_id 
FROM fitas 
WHERE id = 1 AND danificada = 1;
