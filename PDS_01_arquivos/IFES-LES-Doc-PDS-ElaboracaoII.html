07 Atividade: Elaborar Diagrama de Classes de Projeto

Finalidade

  • Elaborar o Diagrama de Classes de Projeto.

Artefatos de Entrada:

  • Diagrama de Casos de Uso
  • Documento de Requisitos

Artefatos de Saída:

  • Diagrama de Classes de Projeto

Papel: Analista e Programador

Ferramentas: Word ou Writer; Astah

Passos:

  1. Abrir arquivo <Sigla do Projeto>-UML_02.asta referente ao último diagrama desenvolvido;
  2. Abrir o Diagrama de Casos de Uso neste arquivo;
  3. Elaborar o Diagrama de Classes de Projeto, tomando como base o Diagrama de Classes de Análise (Modelagem Conceitual), identificando: atributos e navegabilidades;

Instruções para a elaboração do Diagrama de Classes de Projeto:

·       Nomes de atributos começando com letra minúscula;

·       Utilize Integer, Double e Boolean no lugar de int, double e boolean;

·       Padronizar o nome do atributo identificador (quando existir) para id;

·       Utilize o estereótipo <<oid>> para destacar o atributo que identifica exclusivamente o objeto;

·       Inclua navegabilidade para as extremidades obrigatórias, ou seja, com multiplicidades (1 e 1..*). Obs.: caso queira manter atributos não obrigatórios nas classes, não será necessário identificar as navegabilidades;

·       Sugestão: para uma facilidade no projeto do sistema, crie a classe Endereco apenas quando pretender criar a tabela endereço no banco de dados;

Figura 1 - Diagrama de Classes de Projeto para o Sistema de Controle da Videolocadora

  1. Salvar com o nome <Sigla do Projeto>-UML_03.asta
